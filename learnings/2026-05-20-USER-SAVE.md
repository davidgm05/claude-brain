# User Save Trigger 2026-05-20-0939

## Prompt Original
Actúa como arquitecto senior, product manager editorial, ingeniero de automatización, diseñador editorial para Amazon KDP y orquestador multiagente.

OBJETIVO GENERAL

Quiero crear desde cero un sistema automatizado llamado:

KDP Book Factory

El sistema debe investigar ideas de libros sencillos, interesantes y vendibles para Amazon KDP, seleccionar las mejores, investigar a fondo el contenido, crear el libro completo, revisarlo críticamente, mejorarlo, maquetarlo con estilos internos atractivos, generar metadatos comerciales y preparar todo para revisión humana.

IMPORTANTE

No quiero una fábrica de spam.
No quiero libros de baja calidad.
No quiero libros genéricos que parezcan creados por IA sin criterio.

Quiero un sistema editorial automatizado con gates estrictos de calidad.

El sistema debe producir libros útiles, estructurados, interesantes y preparados para que un humano los revise antes de decidir si se publican o no.

NO publiques nada automáticamente en Amazon.
NO subas nada automáticamente a KDP.
NO uses APIs de pago sin confirmación.
NO generes 30 libros desde el primer día.
NO crees contenido basura solo para llegar a 150 páginas.

Todos los libros inicialmente serán:

- Formato: Amazon KDP tapa blanda

PROHIBICIONES Y RIESGOS

El sistema NO debe generar libros que:

- Prometan curar enfermedades.
- Den consejos médicos, legales o financieros de alto riesgo sin fuentes fiables y disclaimers.
- Copien libros existentes.
- Resuman libros protegidos como si fueran sustitutos.
- Usen marcas registradas en títulos de forma problemática.
- Hagan afirmaciones conspirativas como hechos.
- Difundan desinformación.
- Incluyan contenido sexual explícito.
- Incluyan contenido ofensivo, racista, extremista o ilegal.
- Simulen estar escritos por un experto real inexistente.
- Inventen fuentes.
- Inventen datos científicos.
- Rellenen páginas sin valor solo para llegar a 150 páginas.
- Parezcan contenido genérico producido por IA.
- Tengan capítulos que solo existen para inflar el número de páginas.
- No cumplan la promesa del título.

- no rellena páginas solo para llegar a 150

EVALUACIÓN OBLIGATORIA

El agente debe puntuar de 0 a 100:

1. reader_interest_score
¿El libro engancha o aburre?

2. practical_value_score
¿El lector aprende o consigue algo útil?

3. originality_score
¿Tiene enfoque propio o parece genérico?

4. promise_match_score
¿El contenido cumple lo que promete el título?

5. depth_score
¿Tiene profundidad suficiente para no parecer superficial?

6. clarity_score
¿Se entiende bien para el público objetivo?

7. structure_score
¿El orden del libro tiene sentido?

8. example_quality_score
¿Los ejemplos son buenos, concretos y útiles?

9. filler_detection_score
¿Hay poco relleno y poca repetición?

10. amazon_buyer_value_score
¿Un comprador de Amazon sentiría que ha recibido valor?

REGLAS DE APROBACIÓN

El libro NO puede pasar a maquetación si:

- reader_interest_score < 75
- practical_value_score < 75
- promise_match_score < 80
- filler_detection_score < 80
- amazon_buyer_value_score < 75

Si falla cualquiera de esas puntuaciones, el estado debe ser:

NEEDS_VALUE_REVISION

Si el libro es claramente débil, genérico o aburrido, el estado debe ser:

REJECTED_BEFORE_FORMATTING

Si el libro pasa, el estado debe ser:

APPROVED_FOR_FORMATTING

10. ¿Qué ejercicios, tablas, fichas o checklists añadirían valor?

12. ¿Qué haría que el libro tuviera más posibilidades de recibir buenas reseñas?

CRITERIOS DE RECHAZO AUTOMÁTICO

Rechaza una idea o libro si:

- Es demasiado genérico.
- No tiene comprador claro.
- Tiene alto riesgo legal o factual.
- Requiere autoridad médica/legal/financiera que no tenemos.
- Se basa en marcas registradas.
- Parece resumen de otro libro existente.
- No puede llegar a 150 páginas sin valor real.
- El contenido parece repetitivo.
- El contenido parece inventado.
- No se puede verificar.
- El QA score es bajo.
- El Critical Reader considera que el libro no es suficientemente interesante.
- El libro no aporta valor real al lector.
- El libro parece contenido genérico producido por IA.
- El libro tiene capítulos que solo existen para rellenar páginas.
- El libro no cumple la promesa del título.
- El libro no merece pasar a diseño interior.
