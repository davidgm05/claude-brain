# User Save Trigger 2026-07-15-1011

## Prompt Original
# MEJORA OBLIGATORIA DE LOS CONTENEDORES

Una vez reparado el funcionamiento base, mejora profundamente los cuatro contenedores.

No deben parecer cuatro tarjetas iguales con distinto color.

Cada destino debe tener una estructura física, una silueta y una animación propias.

## CONTENEDOR NORMAL

Debe parecer una salida logística estándar:

* Rodillos.
* Cinta secundaria.
* Flecha de expedición.
* Luz verde discreta.
* Marco metálico.
* Señal “DESPACHO”.
* Pequeño contador mecánico.
* Movimiento continuo de rodillos.

Cuando recibe un paquete correctamente:

* Los rodillos se activan.
* El paquete entra horizontalmente.
* Se enciende una luz verde.
* El contador aumenta.
* Suena un mecanismo corto y satisfactorio.

Cuando recibe un paquete incorrecto:

* Los rodillos se detienen.
* Luz roja breve.
* El paquete rebota hacia fuera.

## CONTENEDOR FRÁGIL

Debe parecer una estación acolchada:

* Base acolchada.
* Correas.
* Barras amarillas y negras.
* Icono de copa rota.
* Amortiguadores.
* Bandeja con movimiento suave.
* Señal “MANIPULAR CON CUIDADO”.

Cuando recibe un paquete correctamente:

* La bandeja baja lentamente.
* Las correas se cierran.
* El paquete se deposita con cuidado.
* Se ilumina en ámbar.

Cuando recibe un paquete incorrecto:

* La bandeja tiembla.
* Se activa una alarma corta.
* El paquete vuelve hacia la estación.

## CONTENEDOR DEVOLUCIÓN

Debe parecer una rampa inversa:

* Cinta en sentido contrario.
* Impresora de etiquetas.
* Sello rojo.
* Flechas de retorno.
* Bandeja metálica.
* Señal “DEVOLVER AL REMITENTE”.

Cuando recibe un paquete correctamente:

* Se imprime una etiqueta.
* Aparece un sello rojo.
* La cinta lo devuelve hacia el fondo.
* Suena una impresora térmica.

Cuando recibe un paquete incorrecto:

* La impresora muestra error.
* El paquete es rechazado.
* La luz roja parpadea.

## CONTENEDOR CUARENTENA

Debe parecer una cámara de aislamiento:

* Puerta hermética.
* Marco reforzado.
* Cristal oscuro.
* Vapor.
* Pestillos.
* Luz morada.
* Símbolo de peligro.
* Medidor de presión.
* Señal “AISLAMIENTO”.

Cuando recibe un paquete correctamente:

* La puerta se abre.
* El paquete entra.
* Los pestillos se cierran.
* Sale vapor.
* La luz morada pulsa.
* Puede producirse una reacción distinta según el paquete.

Cuando recibe un paquete incorrecto:

* La puerta se bloquea.
* Alarma.
* Vibración.
* El paquete rebota violentamente.

## DIFERENCIACIÓN VISUAL

Cada contenedor debe diferenciarse mediante:

* Forma.
* Altura.
* Silueta.
* Material.
* Símbolo.
* Color.
* Patrón.
* Animación.
* Sonido.
* Dirección de entrada.

No depender únicamente del borde de color.

## ESTADO NORMAL

Cuando no se está arrastrando ningún paquete:

* Los contenedores deben verse activos, pero no brillar constantemente.
* Mantener pequeñas animaciones ambientales.
* Evitar pulsos fuertes permanentes.

## ESTADO DE ARRASTRE

Mientras se arrastra un paquete:

* Los contenedores disponibles se vuelven ligeramente más legibles.
* No revelar cuál es correcto.
* El destino bajo el puntero se destaca.
* Los demás reducen ligeramente su presencia.
* El resaltado debe usar forma, luz y movimiento, no solo color.

## ESTADO HOVER

Al entrar sobre un contenedor:

* Elevar o activar su mecanismo.
* Mostrar claramente la zona de soltado.
* Aplicar un contorno amplio.
* Cambiar el cursor.
* Añadir una respuesta sonora muy breve.
* No revelar si la clasificación es correcta.

## ZONA DE SOLTADO

La zona de drop debe ser mayor que el dibujo visible.

Añade un área invisible de tolerancia alrededor:

```css
.container-dropzone {
  position: absolute;
  inset: -16px;
}
```

En móvil, ampliar hasta aproximadamente 24 px.

La detección debe ser indulgente y no exigir una precisión absurda.

# MEJORA OBLIGATORIA DEL DRAG AND DROP

El arrastre actual debe sentirse físico, fluido y fiable.

No debe parecer que se mueve un bloque HTML pegado al cursor.

## POINTER EVENTS

Utiliza Pointer Events:

* `pointerdown`
* `pointermove`
* `pointerup`
* `pointercancel`

No mezclar implementaciones separadas de ratón y táctil.

Al comenzar:

```js
packageElement.setPointerCapture(event.pointerId);
```

Al terminar:

```js
packageElement.releasePointerCapture(event.pointerId);
```

Controlar errores si ya no tiene la captura.

## POSICIÓN INICIAL

Guardar siempre:

```js
startPointerX
startPointerY
startTranslateX
startTranslateY
```

No calcular todo únicamente desde `offsetLeft`.

Usar `getBoundingClientRect()`.

## MOVIMIENTO

Actualizar mediante `requestAnimationFrame`.

No cambiar el DOM en cada evento sin control.

Patrón recomendado:

```js
let pendingX = 0;
let pendingY = 0;
let frameRequested = false;

function onPointerMove(event) {
  pendingX = event.clientX;
  pendingY = event.clientY;

  if (frameRequested) return;

  frameRequested = true;

  requestAnimationFrame(() => {
    frameRequested = false;
    updatePackagePosition(pendingX, pendingY);
  });
}
```

## INERCIA VISUAL

Durante el arrastre:

* Inclinar según la velocidad horizontal.
* Elevar ligeramente.
* Separar la sombra.
* Aumentar el tamaño entre 2 % y 5 %.
* Aplicar resistencia según el peso.
* Reducir la inclinación en paquetes pesados.
* Aumentar la rotación en tubos y sobres.

Ejemplo:

```js
const velocityX = currentX - previousX;
const tilt = clamp(
  velocityX * 0.12,
  -maxTilt,
  maxTilt
);
```

Aplicar:

```js
transform:
  translate3d(x, y, 0)
  rotate(tilt)
  scale(1.04);
```

## PESO

Cada paquete debe tener un coeficiente de masa:

```js
dragMass = 0.8;
```

Ejemplos:

* Sobre: 0.65.
* Caja pequeña: 0.8.
* Caja estándar: 1.
* Caja pesada: 1.4.
* Caja de madera: 1.5.
* Tubo: 0.9.

Usar la masa para variar:

* Seguimiento del puntero.
* Inercia.
* Rotación.
* Rebote.
* Duración de retorno.
* Sonido.

Los objetos pesados no deben quedarse retrasados de forma incómoda, pero sí transmitir más resistencia.

## SOMBRA DINÁMICA

Crear una sombra separada:

```html
<div class="package-shadow"></div>
```

Durante el arrastre:

* La sombra permanece más cerca de la cinta.
* Se hace más borrosa al elevar el paquete.
* Se desplaza ligeramente en dirección opuesta.
* Vuelve a unirse al soltar.

No animar un `box-shadow` enorme en cada frame si afecta al rendimiento.

## LÍMITES

No permitir que el paquete desaparezca completamente fuera de pantalla.

Aplicar límites:

```js
x = clamp(x, minX, maxX);
y = clamp(y, minY, maxY);
```

Debe poder acercarse a los bordes, pero mantener siempre una parte visible.

## DETECCIÓN DEL DROP

Usar el centro del paquete y varios puntos de prueba:

* Centro.
* Esquina superior izquierda.
* Esquina superior derecha.
* Esquina inferior izquierda.
* Esquina inferior derecha.

No depender únicamente del puntero.

Ejemplo:

```js
function detectDropTarget(packageRect) {
  const points = [
    [packageRect.left + packageRect.width / 2, packageRect.top + packageRect.height / 2],
    [packageRect.left + 10, packageRect.top + 10],
    [packageRect.right - 10, packageRect.top + 10],
    [packageRect.left + 10, packageRect.bottom - 10],
    [packageRect.right - 10, packageRect.bottom - 10]
  ];

  for (const [x, y] of points) {
    const target = document
      .elementFromPoint(x, y)
      ?.closest('[data-container]');

    if (target) return target;
  }

  return null;
}
```

## MAGNETISMO

Cuando el paquete se aproxima a un contenedor:

* Aplicar una atracción visual suave.
* No moverlo bruscamente.
* No activar la clasificación hasta soltar.

Ejemplo:

```js
if (distance < magnetRadius) {
  x += (targetCenterX - packageCenterX) * 0.08;
  y += (targetCenterY - packageCenterY) * 0.08;
}
```

El magnetismo debe ser más fuerte en móvil.

## RESALTADO DE DESTINO

Mientras se arrastra:

```js
container.classList.toggle(
  'drop-hover',
  container === currentTarget
);
```

Al cambiar de destino:

* Quitar el anterior.
* Activar el nuevo.
* Reproducir sonido suave una sola vez.
* Evitar vibrar continuamente.

## SOLTAR EN DESTINO

Cuando existe un destino:

1. Bloquear nuevos inputs.
2. Animar el paquete hacia el centro del mecanismo.
3. Reducir tamaño según la profundidad.
4. Ejecutar la reacción del contenedor.
5. Validar la clasificación.
6. Mostrar feedback.
7. Retirar el paquete.
8. Cargar el siguiente.

No validar antes de que la animación empiece.

## SOLTAR FUERA

Cuando no existe destino:

* El paquete debe volver suavemente.
* Usar una animación de resorte.
* No teletransportarlo.
* Añadir un pequeño rebote al llegar.
* No aplicar penalización.

Ejemplo:

```css
.package.returning {
  transition:
    transform 420ms cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

## CANCELACIÓN

Gestionar:

* `pointercancel`
* Pérdida de foco.
* Cambio de pestaña.
* Pausa.
* Apertura de manual.
* Fin de turno.

En todos esos casos:

* Cancelar el arrastre.
* Quitar resaltados.
* Liberar captura.
* Volver a posición inicial.
* Restaurar el estado.

## MÓVIL

En móvil:

* Ampliar zonas de drop.
* Reducir inclinación.
* Aumentar magnetismo.
* Mantener el paquete por encima del dedo.
* Evitar que el dedo tape la etiqueta.

Aplicar un desplazamiento vertical:

```js
const touchOffsetY = -50;
```

Solo para punteros táctiles.

## ACCESIBILIDAD

Permitir clasificar también con:

* Clic en contenedor.
* Teclas 1, 2, 3 y 4.
* Foco por teclado.

El drag and drop no debe ser la única opción.

## FEEDBACK HÁPTICO

En dispositivos compatibles:

Al entrar en una zona:

```js
navigator.vibrate?.(8);
```

Al acertar:

```js
navigator.vibrate?.([20, 30, 20]);
```

Al fallar:

```js
navigator.vibrate?.([60, 40, 60]);
```

Respetar el ajuste de vibración.

## RENDIMIENTO

Durante `pointermove`:

* Solo animar `transform`.
* No consultar estilos computados repetidamente.
* Cachear rectángulos de contenedores al empezar.
* Recalcular si cambia orientación o tamaño.
* Evitar crear nodos.
* Evitar partículas.
* Evitar sombras pesadas por frame.

## LIMPIEZA

Al terminar cualquier arrastre:

```js
function cleanupDragState() {
  packageElement.classList.remove(
    'dragging',
    'returning'
  );

  containers.forEach(container => {
    container.classList.remove(
      'drop-hover',
      'drop-candidate'
    );
  });
}
```

Debe ejecutarse siempre, incluso si ocurre un error.

# CRITERIOS DE ACEPTACIÓN DE CONTENEDORES Y ARRASTRE

No des esta parte por terminada hasta que:

* Los cuatro contenedores tengan siluetas diferentes.
* Cada uno tenga una animación propia.
* Cada uno tenga una reacción propia.
* Las zonas de drop sean amplias.
* No se revele la solución.
* El paquete siga el puntero con suavidad.
* La inclinación responda a la velocidad.
* El peso cambie la sensación.
* La sombra transmita elevación.
* El paquete no desaparezca fuera de pantalla.
* El magnetismo ayude sin robar el control.
* Soltar fuera devuelva el paquete suavemente.
* Soltar dentro active una trayectoria física.
* El arrastre funcione con ratón.
* El arrastre funcione con táctil.
* Se pueda clasificar mediante teclado.
* Reiniciar no duplique listeners.
* No haya errores de consola.
