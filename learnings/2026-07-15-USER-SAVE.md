# User Save Trigger 2026-07-15-0826

## Prompt Original
Actúa como un equipo senior formado por:

* Game director especializado en juegos indie.
* Diseñador de sistemas y progresión roguelite.
* Diseñador UX de videojuegos.
* Programador experto en HTML, CSS y JavaScript vanilla.
* Diseñador de sonido y feedback audiovisual.
* Especialista en accesibilidad, responsive y rendimiento web.

Continúa trabajando directamente sobre la última versión del juego **Paquete 13** que ya tienes en el proyecto y que acabas de modificar.

No voy a enviarte nuevamente los archivos porque ya conoces la estructura, el código actual y los cambios realizados anteriormente.

Debes revisar la última versión completa existente en el proyecto y mejorarla sobre esa misma base.

No crees un proyecto nuevo.

No crees una versión paralela.

No sustituyas el juego por una demo simplificada.

No vuelvas a una versión anterior.

No elimines funcionalidades que ya funcionan.

No te limites a explicarme los cambios.

Debes modificar directamente los archivos actuales del proyecto y dejar la última versión completamente funcional.

Antes de hacer cambios:

1. Abre y revisa todos los archivos actuales relacionados con el juego.
2. Comprueba qué sistemas ya están implementados.
3. Detecta errores, comportamientos incompletos y código duplicado.
4. Conserva las mejoras visuales y jugables que ya funcionan.
5. Implementa las siguientes mejoras sobre la última versión.
6. Ejecuta y prueba el juego después de modificarlo.
7. Revisa la consola y corrige todos los errores.
8. Comprueba una partida completa desde el menú hasta la pantalla de resultados.
9. Comprueba el tutorial, el reinicio, la pausa y el guardado.
10. No termines hasta que el juego pueda jugarse correctamente.

OBJETIVO PRINCIPAL

La última versión ha mejorado mucho visualmente, pero todavía parece una interfaz de clasificación en lugar de un videojuego de organización de paquetes con tensión, deducción, movimiento, ambiente y progresión.

El objetivo de esta fase es convertir la versión actual en un juego mucho más vivo, entretenido y adictivo mediante:

* Deducción real.
* Reglas progresivas.
* Paquetes visualmente diferentes.
* Cinta transportadora funcional.
* Herramientas con costes y decisiones.
* Tutorial correctamente colocado.
* Feedback audiovisual satisfactorio.
* Progresión roguelite.
* Eventos de almacén.
* Narrativa ambiental.
* Dificultad progresiva.
* Mayor rejugabilidad.

IMPORTANTE

Aplica todas las instrucciones siguientes directamente sobre la última versión existente del proyecto.

Cuando una funcionalidad solicitada ya exista:

* No la dupliques.
* Revísala.
* Corrígela.
* Complétala.
* Mejora su integración.
* Elimina implementaciones antiguas o incompatibles.

Cuando haya varias versiones de una misma función, conserva una sola implementación limpia y funcional.

Al finalizar:

* Guarda todos los cambios en los archivos actuales.
* Prueba el juego en el navegador.
* Corrige errores visuales y JavaScript.
* Indícame exactamente qué archivos modificaste.
* Resume las funcionalidades implementadas.
* Explica cualquier elemento que no hayas podido completar.
* No pegues todo el código en el chat salvo que sea imprescindible.
