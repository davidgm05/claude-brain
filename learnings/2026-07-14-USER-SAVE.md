# User Save Trigger 2026-07-14-1343

## Prompt Original
Revisa el archivo HTML adjunto de “Paquete 13” y realiza una reparación profunda. No añadas nuevas mecánicas ni más efectos hasta que el juego actual sea estable.

El archivo contiene una mezcla del prototipo antiguo y la nueva arquitectura. Debes limpiar el proyecto, corregir los errores de ejecución y entregar una versión jugable y autocontenida.

OBJETIVO DE ESTA TAREA:

Entregar un único archivo `index.html` que:

* Funcione sin carpetas externas.
* No produzca errores 404.
* No produzca errores en consola.
* No tenga código duplicado.
* No registre listeners múltiples.
* Permita jugar una noche completa.
* Permita usar herramientas, arrastrar, pausar, reiniciar y avanzar.
* Funcione con ratón, teclado y pantalla táctil.
* Sea responsive en escritorio y móvil.

No añadas más contenido hasta estabilizar todo lo existente.

========================================

1. ELIMINAR DUPLICACIONES
   ========================================

El HTML actual contiene:

* CSS antiguo y CSS nuevo.
* Variables CSS definidas varias veces.
* Scripts externos y los mismos scripts incrustados.
* Componentes antiguos y nuevos.
* Handlers inline antiguos.

Haz lo siguiente:

* Elimina todas las etiquetas `<script src="js/...">`.
* Mantén todo el JavaScript en un único bloque al final del documento.
* Elimina todos los `@import` del CSS inline.
* Conserva un único bloque de variables CSS.
* Conserva una única definición para cada selector.
* Elimina el CSS completo del prototipo antiguo cuando exista una versión nueva.
* Elimina funciones, clases y constantes no utilizadas.
* No dejes dos implementaciones del timer, herramientas, drag, UI o estado.

========================================
2. UNA ÚNICA MÁQUINA DE ESTADOS
===============================

Actualmente se mezclan:

* `Game.state`
* `gameState.current`

Utiliza exclusivamente `gameState`.

Estados permitidos:

* BOOT
* MENU
* NIGHT_INTRO
* PACKAGE_ENTERING
* INSPECTION
* DRAGGING
* RESOLVING
* NIGHT_END
* RESULTS
* PAUSED
* SETTINGS

Todo módulo debe consultar:

```js
gameState.current
gameState.isInteractive()
gameState.isRunning()
gameState.transition()
```

Elimina las comprobaciones directas de `Game.state`.

Puedes añadir en `Game` un getter de solo lectura:

```js
get state() {
  return gameState.current;
}
```

Pero no mantengas dos valores independientes.

========================================
3. ELIMINAR HANDLERS ANTIGUOS
=============================

Elimina de los contenedores:

```html
ontouchstart="handleContainerTouch(...)"
onmouseenter="handleContainerHover(...)"
onmouseleave="handleContainerHover(...)"
```

Las funciones `handleContainerTouch` y `handleContainerHover` no existen.

No uses handlers inline.

Registra las interacciones desde JavaScript con:

* pointerenter
* pointerleave
* pointerdown
* pointermove
* pointerup
* pointercancel

El `DragController` debe ser el único sistema de arrastre.

const CONTAINER_TYPE_MAP = {

Prioridad:

```js
QUARANTINE > RETURN > FRAGILE > NORMAL
```

Añade una prueba automática para verificar que cada categoría encuentra su contenedor HTML.

========================================
5. CORREGIR LISTENERS DUPLICADOS
================================

Actualmente existen listeners de teclado en varios módulos.

Debe haber un solo controlador global:

```text
B = báscula
E = escáner
L = UV
1 = normal
2 = frágil
3 = devolver
4 = cuarentena
5, 6, 7 = power-ups
Escape = pausa
```

Elimina nombres intermedios incorrectos como:

```js
box-cutter
ear-defenders
uv-light
```

Usa directamente:

```js
scale
scanner
uv
```

Todos los módulos deben tener protección:

```js
if (this.initialized) return;
this.initialized = true;
```

No vuelvas a registrar listeners al reiniciar una noche o volver al menú.

========================================
6. CORREGIR EL TIMER
====================

Debe existir un único timer HTML:

```html
<div id="timer-display">
  <svg>...</svg>
  <div class="timer-progress">
    <div class="timer-progress-fill"></div>
  </div>
  <span class="timer-time">30s</span>
</div>
```

Elimina `Timer.getVisual()` si genera otro timer diferente.

Actualiza:

```js
const fill = timerEl.querySelector(".timer-progress-fill");
fill.style.width = `${percent * 100}%`;
```

No modifiques el ancho de `.timer-progress`.

El timer debe:

* Iniciarse al entrar en `INSPECTION`.
* Continuar durante `DRAGGING`.
* Pararse durante `RESOLVING`.
* Pararse durante `PAUSED`.
* Pararse durante animaciones de entrada.
* Reiniciarse en cada paquete.
* No crear intervalos.
* Usar el delta time del game loop.
* No continuar tras volver al menú.
* Emitir el evento de tiempo agotado una sola vez.

Colócalo dentro del HUD y elimina estilos contradictorios con `position: fixed` en el centro.

========================================
7. CORREGIR PAUSA
=================

Al pausar:

* Guarda el estado anterior.
* Para el timer.
* Pausa el game loop.
* Cancela un drag activo.
* No clasifica el paquete.
* Muestra la pantalla de pausa.

Al reanudar:

* Vuelve al estado anterior seguro.
* Si estaba arrastrando, vuelve a `INSPECTION`.
* Ajusta `lastTime` para evitar saltos.
* Reanuda el timer con el tiempo restante.

Al cambiar de pestaña:

* Pausa el juego.
* No reanudes automáticamente al volver.
* Espera a que el jugador pulse continuar.

Nunca deben escribir directamente sobre `_currentPackage` desde fuera de `Game`.

========================================
9. CORREGIR HERRAMIENTAS
========================

Las herramientas solo funcionan cuando:

```js
gameState.current === GameState.INSPECTION
```

o cuando se permita explícitamente su uso.

Al usar una herramienta:

* Bloquea temporalmente otros usos.
* Ejecuta una sola animación.
* Añade una sola pista al panel.
* No duplica pistas.
* No aumenta inestabilidad por detectar anomalías.
* No modifica un paquete que ya fue clasificado.
* No conserva resultados del paquete anterior.

En cada paquete nuevo:

```js
Tools.resetInspection();
```

En cada noche nueva:

```js
Tools.resetCooldowns();
```

El cooldown debe detenerse durante pausa y resultados.
